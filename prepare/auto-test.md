# Pre-task thinking

Auto-test does unified automated testing via a CSV layered pipeline. It auto-selects scenario sources from project state (specs / coverage gaps / code exploration), then: discover → plan → build CSV → write tests (parallel) → execute → diagnose failures (parallel) → iterate → report.

Layers L0→L3 run in order (fail-fast), scenarios within a layer run in parallel. `--max-iter 1` = single-pass generation; default = full iteration cycle.

## Mode determination / smart routing

$ARGUMENTS must provide a phase number. Routing is auto-detected from project state:

| Priority | Condition | Route |
|----------|-----------|-------|
| 1 | active session exists (state.json status=running) | Resume |
| 2 | `--re-run` flag + last run had failures | Re-run |
| 3 | Spec package exists (REQ-*.md) | spec source |
| 4 | Nyquist gaps exist (verification.json) | gap source |
| 5 | default | code source |

## Flags

| Flag | Effect | Default |
|------|--------|---------|
| `-c N` | continue from iteration N of an existing session (skip setup) | — |
| `-y` | skip confirmation prompts for artifact registration and issue creation | false |
| `--max-iter N` | max outer iterations; set 1 for single-pass generation | 5 |
| `--layer L` | start from / limit to the specified layer (L0\|L1\|L2\|L3) | — |
| `--dry-run` | only generate the test plan, do not execute | false |
| `--re-run` | only re-run previously failed/blocked scenarios | false |

## Input and boundaries

- phase number is required (missing when no active session → E001)
- **Output boundary**: all file writes must land in `{run_dir}/outputs/` or `.workflow/state.json`. Source modification is limited to test files generated by this command; never modify existing production source

## Pre-load (all optional, continue if missing)

1. **Test specs + tools**: `maestro load --type spec --category test` — load testing conventions (framework, patterns, naming), apply to all generated tests
2. **Coding specs**: `maestro load --type spec --category coding` — understand coding patterns to target tests precisely
3. **Role Knowledge**: browse with `maestro search --category test`, load relevant entries with `maestro load --type knowhow --id <id1> [id2...]`

## Invariants

1. **Test code isolation** — generated tests must be written to a test directory following project conventions; never modify production source to make tests pass
2. **RED-GREEN integrity** — tests must first be written to fail (RED), then verified to pass against the implementation (GREEN); never write trivially-passing tests or empty assertions
3. **Strict layer ordering** — L0→L3 execute in order and fail-fast; never skip a layer or run L2 before L1 passes
4. **Existing tests are sacred** — never modify/delete/overwrite existing test files; new tests are added incrementally only
5. **Convergence needs evidence** — pass rate and confidence score must be computed from actual execution results; never inferred without running
6. **Bounded iteration** — outer iterations do not exceed `--max-iter` (default 5); inner test_defect fixes escalate after no more than 3 per scenario
7. **spec source mandatory traceability** — when route=spec, every test scenario must trace back to some REQ-*.md requirement; untraceable tests are flagged
