## v0.5.53

### Overview

本版本把 Session/Run 推进到 `session/1.3` 与 `command-run/1.3`，补齐 topic identity、reuse assessment、paused recovery、mutation receipt、machine response 和 artifact TOCTOU 边界。同时重新划分 Maestro 的三层入口：`/maestro-companion` 负责机械明确的轻量直接执行，`/maestro-next` 只做复杂度判断与渠道推荐，`/maestro` 负责需要多个生命周期阶段的 manual/Ralph 编排。原 first-tier `quick` step 与 `/maestro-quick` 已退休。

**Full Changelog**: https://github.com/catlog22/maestro-flow/compare/v0.5.52...v0.5.53

### 🚀 Features

- **Session/Run 1.3 contract** — writer 升级到 `session/1.3` / `command-run/1.3`，reader 保持对 1.0–1.3 的兼容；新增 topic identity、reuse assessment 与更完整的 machine operation surface（`5f5bba60`, `42aa655b`, `31e53a58`）。
- **Companion minimal Run channel** — Companion 使用 chainless create/complete 生命周期直接执行任务并记录非正式 evidence，不再伪装成知识加载器或 plan/execute 缩短链（`df91d7f9`, `ab7a6bc2`）。
- **Three-level routing model** — Next 成为纯 router，按认知复杂度推荐 Companion、标准单 Run 或 Maestro；Maestro 支持 manual/Ralph 双引擎并阻止轻量任务过度编排（`ab7a6bc2`, `d11699b8`, `7010df33`）。
- **Release contract gates** — 新增 Session/Run contract parity 与 release-machine smoke，覆盖 schema、machine handlers、accept-reuse applied/replayed 语义及 prepublish 顺序（`31e53a58`, `21481862`）。

### 🐛 Bug Fixes

- **Durable transition and replay** — 加固 SessionStore 锁恢复、可重试 mutation receipt、complete replay byte fence、recall confirmation 与 Windows transient lock handling（`160a8f75`, `2d646a9a`, `c955a602`, `016c1407`）。
- **Paused recovery authority** — paused Session 恢复改为 canonical、显式且可审计的路径，不再隐式分配新 Run 或通过历史相似结果获得 mutation authority（`03b6c58d`, `1024a99a`）。
- **Artifact boundary hardening** — artifact template 扫描、symlink/junction 与 TOCTOU 重新验证 fail closed，release machine 使用 fresh built CLI 验证（`0649c0cf`, `21481862`）。
- **Quick retirement cleanup** — installer 升级时清理遗留 `prepare/quick.md`、`workflows/quick.md` 与生成的 `maestro-quick.md`，packaged consumer 不再暴露退休 step（`92bcd3ba`）。

### 🔧 Refactors

- **Retire first-tier quick** — 删除 quick prepare/workflow contract；所有轻量意图直接映射到 `/maestro-companion`，`quick` 仅作为自然语言分类关键词或独立 quality level 保留（`92bcd3ba`）。
- **Intent and argument separation** — `--intent` 仅作为 Session metadata；命令实际输入通过 repeatable `--arg` 或 `-- <args...>` 传递，Companion 同时传递 metadata 与 required command input（`d519b5d2`, `92bcd3ba`）。
- **Cross-platform mirror alignment** — Claude、Codex、Agy、Agents 的 Run ownership、agent lifecycle、tool invocation 与 generated mirror lint 对齐统一 contract（`d3ae7f7a`, `92bcd3ba`）。

### 📝 Documentation

- 新增完整 Session/Run architecture 文档，并同步 run-mode、command/skill authoring 与 Codex adapter 约束（`5b000281`, `92bcd3ba`）。
- 更新 Maestro、Next、Companion、Init、Overlay、Help、inventory、Quick Start 与双语 docs-site，移除 Next 仍隐式执行 analyze → plan → execute 的旧叙述（`92bcd3ba`）。
- 更新 migration reference：旧 `/maestro-quick` 使用 `/maestro-companion`，需要正式 artifact/gate 时使用标准 Run 或 `/maestro`（`92bcd3ba`）。

### Compatibility Notes

- `/maestro-quick`、`prepare/quick.md` 与 `workflows/quick.md` 已退休。轻量任务改用 `/maestro-companion "<intent>"`；不确定复杂度时使用 `/maestro-next "<intent>"`。
- `/maestro-next` 不再代表固定 quick chain，也不会隐式展开 analyze → plan → execute。
- `--intent` 是 Session metadata，不满足 command contract 的必填输入；调用方必须使用 `--arg` 或 `--` 传递命令参数。
- 新写入使用 `session/1.3` 与 `command-run/1.3`；旧 1.0–1.2 数据继续兼容读取。
