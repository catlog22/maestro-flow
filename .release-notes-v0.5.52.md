## v0.5.52

### Overview

本版本完成 Session/Run 1.2 的运行时闭环：`run complete` 不再只写入完成状态，而是返回统一的 `run-response/1.0` machine envelope，自动给出当前 Session 的下一动作、简要执行指南、命令 args、requirements、已解析输入和缺失输入。与此同时，Session 复用从模糊自动恢复改为 Unicode-safe exact identity + advisory recall，历史相似 Session 只能经 confirmation token 显式 fork/import/new，避免误续跑与跨意图污染。

**Full Changelog**: https://github.com/catlog22/maestro-flow/compare/v0.5.51...v0.5.52

### 🚀 Features

- **Machine-readable completion and next action** — 新增 `run-response/1.0`，统一 `complete`、`next`、`brief` 等命令的 machine/display 边界；完成后直接返回 `next_action`、简要 command/skill 信息与可执行建议（`f2c5be71`）。
- **Args and requirement discovery** — `command-run/1.2` 将 command args、input requirements、resolved inputs、missing inputs 与执行 contract 一并写入 birth packet，LLM 可在派发前判断还需要哪些输入（`f2c5be71`）。
- **Safe Session recall and reuse** — 新增 Unicode-safe `intent-identity/1.0`、exact candidate 与 historical candidate 分层；历史相似结果保持 `automatic=false`，仅能通过有时效、单次消费的 confirmation token 执行 fork/import/new（`f2c5be71`）。
- **Session lifecycle operations** — 增加 recall、recall-confirm、fork、import、new、rebind、resolve、resume 等显式操作，并提供 lineage、transition receipt 和审计证据（`bb783cad`, `f2c5be71`）。
- **Session/Run wiki and dashboard integration** — 搜索、wiki topology、team lifecycle 与 dashboard 路由对齐 Session/Run 数据模型，支持恢复与关联信息展示（`8a0ce6e2`, `e37b40a6`）。

### 🐛 Bug Fixes

- **Durable SessionStore** — 通过 durable lock、原子 mutation、crash recovery、transition receipt 与 fencing 防止并发写入、重复确认和半完成状态（`f2c5be71`）。
- **Fail-closed recovery** — `resolve`、`resume`、`rebind` 对歧义、过期 token、lineage 不一致和缺失证据显式阻断，不再静默选择候选 Session（`f2c5be71`）。
- **Canonical-first Ralph execution** — Ralph 优先读取 Session/Run canonical state，legacy `ralph-meta.json` 仅保留兼容读路径，避免状态双写和 schema 漂移（`f2c5be71`, `af889938`）。
- **Contract compatibility** — 新增 schema compatibility、contract v2 opt-in 和 migration 覆盖，保持旧 `session/1.0` / `session/1.1` 数据可读（`f2c5be71`）。

### 🔧 Refactors

- **Cross-platform agent contracts** — Codex agent override、TOML 生成和 Claude/Codex/Agy/Agents mirror pipeline 对齐新的 Session/Run contract（`e93f8e83`, `f4344803`）。
- **Run context and team lifecycle** — run context、checkpoint、issue store 与 team session lifecycle 统一使用 Session/Run 路径和状态边界（`bb783cad`, `e37b40a6`）。

### Compatibility Notes

- 集成方应优先消费 `run-response/1.0` 的 machine payload；display 文本仅用于人类快速阅读。
- `run complete` 的后续建议不授予 mutation authority；涉及 historical recall 时必须使用 CLI 返回的 confirmation token。
- live exact resume 由 Unicode-safe intent identity 与 SessionStore 状态共同决定；intent slug 不再是恢复依据。
- 新写入使用 `session/1.2` 与 `command-run/1.2`；旧 schema 继续兼容读取，contract v2 需显式 opt-in。
