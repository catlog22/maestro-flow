## v0.5.51

### Overview

本版本完成 Maestro-Flow 从分散命令状态到统一 Session/Run 生命周期的系统性迁移，并同步升级 Codex Multi-Agent V2、Explore 并行搜索、知识生命周期、跨平台安装与文档站。核心产品范围 `v0.5.50..130e9679` 共 104 个提交：20 个 feature、38 个 fix、22 个 refactor、10 个 docs、8 个 chore、1 个 test、1 个 merge，以及 4 个 Odyssey 执行提交；发布准备另加入版本/docsite 同步、fresh-build 修复、Codex 镜像同步与 Release Note 定稿 4 个提交。

**Full Changelog**: https://github.com/catlog22/maestro-flow/compare/v0.5.50...v0.5.51

---

### Highlights

- **Session/Run 成为唯一生命周期模型**：统一 `prepare → create → brief → check → complete`，新增 `run next` 步进、显式 ASCII Session ID、progressive injection、finish 收口清单和标准 artifact boundary。
- **Codex Multi-Agent V2 原生化**：接入 collaboration schema、TOML Agent 生成、平台 tool profile、`update_plan` 任务跟踪与 Goal 隔离，生成镜像统一受 lint/check 约束。
- **Explore 重构为证据驱动的并行 Batch**：默认并行、流式结果、repository map、通用 agent loop、集中式 `max-turns`、transport-only proxy fallback 与截断续写。
- **知识系统原子化与可追溯**：Session/Run 搜索拓扑、知识生命周期、原子写入、SQL 注入与嵌套事务修复、损坏索引显式提示全部收敛。
- **命令拓扑与文档站同步收敛**：退役 chains 与旧命令，迁移到 prepare/workflows/ref entry step；docsite 新增 v1/v2 切换、v2 合并命令模型与统一配置指南。

### 🚀 Features

#### Session/Run lifecycle

- **全量 Session/Run 迁移** — 命令、workflow、Ralph 与 team pipeline 统一到 Session authority，移除 scratch / milestone / private status truth source（`84ae24f8`, `6640eb87`, `725e40c4`）。
  - `scope`: `src/run/`, `src/ralph/`, `workflows/`, `prepare/`, `.claude/`, `.codex/`
- **`run next` 薄步进器与 `brief` 单注入点** — 出生包携带 upstream、gates、handoff 与 next 指针，Ralph 降为适配层（`f82458dc`, `ac60b79a`, `9341b3ac`）。
  - `scope`: `src/run/next.ts`, `src/run/inject.ts`, `src/run/chain.ts`, `src/ralph/`
- **显式 Session 命名与完成收口** — `run create --session` 强制可读 ASCII slug；`run check` 全绿后注入 handoff、knowledge、conflict 与 verdict 清单（`24501d28`, `084b3f57`）。
- **Ralph CLI 动词收敛** — orchestration 元数据、lease、decision、retry 与 chain 变形统一到 Session v1.1（`62bec6fd`）。

#### Codex and multi-agent platforms

- **Codex V2 collaboration schema** — 原生支持 V2 collaboration tool surface，并通过 `.codex.md` / prepare-workflow override 注入平台差异（`073522d9`, `586088af`）。
- **Codex Agent TOML 自动生成** — 从 Claude agent source 生成 TOML，并加入一致性 lint（`d424c946`, `13191403`）。
- **跨平台工具映射** — 新增 Codex profile、字段映射、runtime conversion 与 Pi teammate/todo profile（`836de74b`, `57bef7b1`）。
- **任务与 Goal 语义分离** — Codex Todo 映射迁移到 `update_plan`，Goal 仅在显式请求时创建（`57b358a6`, `ed71aa86`）。

#### Explore and search

- **并行流式 Explore** — 默认并行执行、可配置并发、流式返回、session trace 与 stopReason 续写（`8d627ee5`）。
- **MOA / repository-map / tools 重构** — 引入 repository map、通用 agent loop 和集中化 `DEFAULT_EXPLORE_MAX_TURNS`，降低 Batch 重复扫描成本（`18783c43`, `bdd75f74`, `69244003`）。
- **代理 transport fallback** — 仅网络传输失败时绕过失效代理直连，HTTP 业务错误不误重试（`f9186f75`）。
- **证据可靠性加固** — 修正轮次计数、工具协议、结果截断与 ONNX 告警噪声（`e44bf4b1`）。

#### Installation and docs-site

- **安装加法语义与并发安全** — 安装锁、manifest 原子写与 `knownComponentIds` 防止增量安装覆盖既有能力（`628731a4`）。
- **docsite v1/v2 切换** — `VersionContext` 持久化版本选择，v2 inventory 将 66 个旧入口收敛为 19 个合并命令（`eb24a55b`）。
- **统一配置指南** — 汇总 CLI、API、MOA、workspace、hooks、overlay、embedding、环境变量与 spec 配置（`6c2682a0`）。

### Compatibility Notes

- 已废弃的 milestone / scratch / chains / private Ralph status 路径不再是权威状态源；集成方应读取 `.workflow/sessions/{session_id}/` 及 Run 返回的 `run_dir` / `upstream`。
- 被 orchestrator 派发且 birth packet 已包含 `run_id` / `run_dir` 时，executor 不得再次调用 `maestro run create`。
- Codex task tracking 使用 `update_plan`；Goal 工具不再承担普通任务跟踪职责。
- `maestro explore` 的 `--max-turns` 语义和默认轮数已集中化；依赖旧轮次常量的封装需更新。
- 文档站默认展示 v2 合并命令模型，但保留 v1 切换用于旧入口查询。
